Datasets#

The platform supports datasets which are meant to help users do version control, share and manage different datasets. It integrates seamlessly in the existing ecosystem.

How to use?

Datasets tab can be found under the Developer tab. There are multiple functionalities that can be performed for a dataset like creating, adding files, downloading, archiving, deleting, publishing etc. Users have to write code in Notebooks for each of the functionalities listed above. To make it easier for the users, we have provided code snippets in Notebooks under Code Snippet Explorer -> Datasets. Snippets can be easily dragged-and-dropped for usage.     1. By uploading local files: path refers to the relative path of the

  1. By uploading S3 files: source_url for S3 files is in the format:

    s3://bucket-name/file-path

  1. Uploading S3 files from the non-default S3 account: source_url for S3

    files is in the format: s3://bucket-name/file-pathaws_access_key, aws_secret_key, & region are the AWS credentials required to access the S3 files of the account.

  1. Download a dataset

    To download a dataset, use :
dataset = Dataset.get(dataset_id="c6b147a022274092bc3e1b75f5f09d7a") # get ID from GUI
local_read_only = dataset.get_local_copy() # Gets read-only copy of dataset to local cache. Path returned in local_read_only.
local_mutable = dataset.get_mutable_local_copy(path = '') # Gets writable copy of dataset to path specified. Target folder returned in local_mutable.

  1. Archive a dataset

    Datasets can be archived from the UI, where they can be then viewed under the archived section & can be restored

  1. Delete a dataset

    To Delete: 
Dataset.delete(dataset_id="c6b147a022274092bc3e1b75f5f09d7a") # get ID from GUI

  1. Publish a dataset

    Publishing a dataset means making it non-editable
dataset = Dataset.get(dataset_id="c6b147a022274092bc3e1b75f5f09d7a") # get ID from GUI
dataset.publish()
